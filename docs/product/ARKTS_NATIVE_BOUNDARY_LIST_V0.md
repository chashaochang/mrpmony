# ArkTS 页面与 Native 边界清单 v0

项目：MRP 模拟器鸿蒙化  
角色：RD Frontend  
范围：FE-003

## 1. 可以留在 ArkTS 的逻辑

### 1.1 页面与路由
- 应用列表页导航
- 导入页导航
- 运行页导航
- 页面返回与确认弹窗

### 1.2 纯展示逻辑
- 列表渲染
- 空态展示
- 加载态展示
- 导入结果展示
- 运行状态展示
- 错误 Banner / Dialog 展示

### 1.3 轻量交互逻辑
- 点击应用进入运行页
- 点击导入按钮进入导入页
- 点击重试/重启/退出
- 页面 show/hide 生命周期事件转发

### 1.4 ViewModel 级状态整理
- 把 Service 返回的数据映射为页面展示态
- 按 phase 切换不同组件
- 管理页面局部临时状态

## 2. 必须走 Adapter / Native 的逻辑

### 2.1 导入链路
- 文件选择器底层接入
- URI / 沙箱路径转换
- 文件合法性校验
- 包解析
- 文件复制 / 解压 / 注册
- 导入失败原因原始采集

### 2.2 应用元数据链路
- 已安装应用扫描
- 图标与元数据读取
- 应用可运行性判定
- 本地注册表 / 索引读取

### 2.3 运行链路
- 创建运行会话
- 渲染容器绑定
- surface / XComponent / native handle 管理
- 输入事件注入
- 音频输出
- 暂停 / 恢复 / 停止运行
- 异常退出与 native 层资源释放

### 2.4 持久化与底层能力
- 存档路径管理
- 配置落盘
- 日志写入
- 权限与沙箱细节
- 生命周期与系统事件底层适配

---

## 3. 页面层禁止下沉的内容清单

### 3.1 禁止进入页面的 native 细节
- native error code
- session handle
- surface id / window id
- 线程 id
- 原始路径
- 文件描述符
- 原始导入阶段枚举
- 原始运行时回调对象

### 3.2 禁止进入页面的实现逻辑
- 文件复制与解压
- 包头解析
- 存档目录拼接
- 输入协议映射实现
- 音频设备初始化
- 渲染线程管理
- native 生命周期恢复逻辑

### 3.3 禁止页面直接做的调用
- 页面直接调 NAPI 原始方法
- 页面直接管理 native listener
- 页面直接订阅 native 高频事件流
- 页面直接判断兼容性底层规则

---

## 4. 页面层允许拿到的唯一形态

页面只允许拿到：
- `AppItemVM`
- `PackagePreviewVM`
- `RuntimePhase`
- `UiError`
- `canRestart / canExit` 这类布尔控制量

页面不允许拿到：
- 原始导入结构
- 原始会话结构
- 原始渲染对象
- 原始路径对象

---

## 5. 各页面边界结论

### 5.1 AppListPage
ArkTS 保留：
- 列表渲染
- 空态/错误态
- 页面跳转

必须下沉：
- 扫描应用
- 读取应用元数据
- 判断是否可运行

### 5.2 ImportPage
ArkTS 保留：
- 选择文件后的流程编排
- 预览展示
- 导入成功/失败展示

必须下沉：
- 文件选择器底层接入
- 包解析与合法性验证
- 导入执行

### 5.3 RunPage
ArkTS 保留：
- 容器占位与工具栏
- 状态切换展示
- 重启/退出按钮

必须下沉：
- 运行会话创建
- 渲染绑定
- 输入/音频/资源释放

---

## 6. 边界原则

1. ArkTS 负责用户意图与页面编排。  
2. Service 负责业务组织与错误归一。  
3. Adapter 负责 native 能力收口。  
4. 页面层不得因省事而直接接 native 原始对象。  
5. native 命名变化只能影响 Adapter，不允许扩散到页面层。  

---

## 7. 结论

POC 阶段边界已经固定：
- ArkTS 做页面、组件、状态编排
- Adapter / Native 做导入、运行、渲染、输入、音频、存档、路径、日志
- 页面层严格禁止下沉到底层实现细节
