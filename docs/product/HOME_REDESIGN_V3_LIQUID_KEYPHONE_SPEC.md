# Home Redesign V3: Liquid Keyphone

目标：这版不是在旧首页上微调，而是把首页重新定义成「MRP 按键机壳子的本地启动器」。

## 这版要解决的问题

- 旧版顶部信息太多，列表作为主任务被往下挤。
- 旧版只是普通移动端列表，缺少 MRP/按键机识别度。
- 旧版玻璃感主要来自底部 HdsTab，页面主体没有统一的液态玻璃语言。
- 旧版游戏和应用虽然能筛选，但视觉上不像两个类型。
- 列表右侧和文件名需要固定安全区域，不能出现宽度溢出或裁切。

## 屏幕结构

- 顶部只做一个低高度液态玻璃信息层：标题、数量、继续。
- 搜索和筛选独立成两个轻玻璃控件，不再加摘要行、分组标题、营销说明。
- 列表从首屏中段直接开始，保持 7 个左右条目的首屏露出。
- 底部 HdsTab 悬浮在列表上方，列表内容继续延伸到其后面。

## 视觉语言

- 背景：冷白到浅青灰，不使用黄色、紫蓝大渐变或厚重卡片。
- 主体：半透明玻璃层 + 白色高光描边 + 很弱的阴影。
- 列表行：不是实心卡片，使用薄玻璃行容器，圆角控制在 24px 内。
- MRP 气质：左侧使用小型「MR / APP + 三个按键点」标识，暗示按键机，不做 NES 游戏机风格。
- 类型色：游戏偏冷青，应用偏低饱和绿，只做辅助识别。

## 交互规则

- 底部三项：应用、最近、设置。
- 导入入口：右下迷你悬浮加号，和 HdsTab 同级，不占列表布局高度。
- 筛选：全部、游戏、应用；切换后列表直接换内容，不出现额外说明区。
- 行点击：直接进入运行页；更多操作以后放详情或长按，不在主列表堆按钮。

## 实现约束

- HarmonyOS 状态变量继续使用 StateV2：`@ComponentV2`、`@Param`、`@Local`、`@Event`、`@Monitor`。
- HdsNavigation 负责页面跳转。
- 首页底部继续使用官方 HdsTabs floating tab bar：`barPosition(BarPosition.End)`、`vertical(false)`、`barOverlap(true)`。
- 不给 HdsTab 外层额外加不透明背景，不给列表预留一整块空白底座。

## 验收标准

- 第一屏至少 6 个完整或接近完整条目。
- 右侧「游戏 / 应用 / 最近」不裁切。
- 文件名长时只在主信息区域省略，不顶到右侧标签。
- 应用筛选下必须能看到 APP 标识和应用列表。
- 页面一眼不能像模板化 AI demo。
