# 项目概览

## 项目名
MRP 模拟器鸿蒙化

## 当前目标
将现有 MRP 模拟器能力迁移/重构到 HarmonyOS 终端侧，先完成一轮技术可行性验证，并建立后续产品化的工程基础。

## 当前结论
- 可以推进，但先按技术验证项目立项
- 一期不做大而全，只做最小闭环
- 当前优先路线：ArkTS 壳 + C++ 核心 + NAPI 粗粒度桥接

## 一期成功标准
- 能导入 MRP 包
- 能启动代表性样例
- 具备基础渲染、输入、音频、退出、存档能力
- 能做基础日志与问题定位
- 能基于样例集形成兼容性结论

## 当前缺口
- 现有 MRP 模拟器资产位置尚未接入本仓库
- 未确认现有内核语言、渲染链路、音频链路、文件系统模型
- 未建立 HarmonyOS POC 工程

## 建议里程碑
- M0：资产盘点与技术摸底
- M1：HarmonyOS POC 工程骨架
- M2：单样例跑通
- M3：首批样例验证
- M4：进入二期规划
