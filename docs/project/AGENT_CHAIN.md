# MRP 模拟器鸿蒙化 - 多 Agent 推进链路

## 目标

围绕“MRP 模拟器鸿蒙化”建立一个可持续推进的多 agent 协作链路，避免各说各话。

## 第一阶段：方案收敛链
- PM：目标、MVP、场景、验收、待确认项
- Architect：边界、模块、端侧/Native/服务侧分工、风险
- HarmonyOS Senior Engineer：HarmonyOS 可行性、ArkTS/NAPI/C++ 路线、工程结构
- RD Lead：立项判断、一期聚焦、资源与风险、下一轮拆工

## 第二阶段：设计与实现拆解链
- Designer
- HarmonyOS Dev / Frontend
- Backend Dev（按需）

## 第三阶段：验证与交付链
- RD QA
- Main 汇总

## 当前建议顺序
1. PM
2. Architect
3. HarmonyOS Dev
4. RD Lead
5. Designer
6. HarmonyOS Dev / Frontend
7. Backend Dev（如需要）
8. RD QA
