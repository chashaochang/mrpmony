# HOS-007 MRP HarmonyOS POC 第二周接入前置条件清单（真实内核适配前）

## 目标
在进入真实 MRP 内核适配前，先把必须确认的信息、adapter 接入约束、禁止跳过的 gate 一次性列清，避免第二周直接陷入接口返工。

---

## 一、必须拿到的现有内核信息

### 1. 内核基础信息
- 内核语言：C / C++ / 其他
- 当前仓库位置与构建方式
- 是否已有可独立运行的核心库
- 是否存在平台相关代码分支

### 2. 包/资源模型
- `.mrp` 包格式说明
- 入口文件/入口函数约定
- 资源解包是否在内核内部完成
- 是否依赖固定目录结构

### 3. 渲染模型
- 输出格式：RGBA / RGB565 / 其他
- 输出粒度：整帧 / 脏矩形 / draw call
- 分辨率与缩放约束
- 是否已有 frame buffer 概念

### 4. 输入模型
- 支持的输入类型：按键 / 触摸 / 文本输入
- 现有 keyCode 映射表
- 长按、连按、方向键重复规则
- 是否区分软键/菜单键/返回键

### 5. 生命周期与运行时模型
- 启动、暂停、恢复、退出的真实状态机
- 主循环归属：内核线程 / 外部驱动
- 是否允许外部轮询拉帧
- pause/resume 是否已有实现

### 6. 文件系统与存档模型
- 运行目录依赖
- 存档路径规则
- 是否写临时文件
- 是否依赖相对路径/当前工作目录

### 7. 音频模型
- 是否必须同步启动音频
- 输出类型：PCM / callback / push buffer
- 是否有音画同步耦合
- 首周是否允许静音 stub

### 8. 线程与内存模型
- 内核是否自建线程
- 哪些对象线程不安全
- 帧缓冲所有权归谁
- 资源释放顺序要求

### 9. 错误与日志模型
- 是否已有错误码
- 是否已有日志接口
- 崩溃/异常时可否给出最小诊断信息

### 10. 许可证与第三方依赖
- 是否依赖第三方图像/音频/压缩库
- 目标库是否可用于 HarmonyOS 构建
- 是否存在许可证风险

---

## 二、Adapter 层接入前必须满足的约束

### 1. 不允许 ArkTS 直接适配内核细节
- 历史内核差异只允许落在 `adapter/existing_core/`
- ArkTS 只面向冻结后的 facade

### 2. Adapter 只做三类事
- 参数转换
- 状态映射
- 平台差异吸收

### 3. Adapter 不做的事
- 页面状态管理
- 业务配置拼装
- UI 逻辑
- 异步调度策略定义

### 4. 接入前要先定义清楚
- 内核 init 所需参数
- 内核 load/start/pause/resume/release 顺序
- 输入事件转换表
- 帧数据输出结构
- 错误码映射规则

---

## 三、进入真实内核适配前的硬门槛（Gate）

### Gate-1：NAPI 基础链路已稳定
满足：
- `init()` 真机成功
- 模块可重复装载
- 无明显构建/链接不稳定问题

### Gate-2：POC 拉帧链路已稳定
满足：
- `pullFrame()` 已能稳定返回测试帧
- ArkTS 页面显示链路无黑屏/花屏
- 全帧回传性能至少可接受用于 POC

### Gate-3：输入闭环已稳定
满足：
- `sendInput()` 能穿透到 native stub
- 页面到 native 的 key/touch 路径已固定

### Gate-4：生命周期链路已固定
满足：
- `pause()/resume()/release()` 调用时机已冻结
- 页面二次进入不会残留脏状态

### Gate-5：现有内核信息已补齐
满足：
- 上述“必须拿到的现有内核信息”至少 80% 已确认
- 剩余不确定项不影响首轮适配切入

---

## 四、哪些没确认前不能进入真实接入
- 不知道内核真实启动顺序
- 不知道帧输出格式
- 不知道输入 keyCode 对照关系
- 不知道 pause/resume 是否可重入
- 不知道内核是否依赖线程上下文
- 不知道文件路径/工作目录要求

---

## 五、需要谁补齐信息
- **内核维护人**：补内核接口、状态机、渲染/输入模型
- **architect**：拍板 facade 与 adapter 最终边界
- **QA**：在接入前不介入；等 stub 链路稳定后开始接设备验证模板

---

## 六、第二周开工判定
满足以下条件才进入真实内核适配：
1. HOS-001 ~ HOS-006 已完成
2. HOS-007 清单中的关键信息已补齐
3. 当前 v0 facade 不再改方法名
4. 可明确知道 adapter 第一刀先接哪里（建议：init/load/start/pullFrame）
