# 鸿蒙技术路线（第一轮）

## 当前建议
优先采用：
- ArkTS 做应用壳
- C++ 复用/承载模拟器核心
- NAPI 做稳定、粗粒度桥接

## 优先确认的问题
- 现有模拟器核心语言
- 渲染输出模型
- 输入模型
- 音频链路
- 文件系统与存档模型
- 是否已有可复用 native 核心

## 纯 ArkTS 适合的部分
- 页面壳
- 设置与配置
- 导入页与应用列表
- 调试开关与日志展示
- 一部分轻量状态管理

## 需重点评估 NAPI/C++ 的部分
- 模拟器核心
- 高频渲染
- 音频底层
- 大量二进制处理
- 已有 C/C++ 能力复用

## 官方文档优先检查主题
- Stage Model / UIAbility 生命周期
- ArkUI 与绘制能力
- 输入事件
- 文件管理与应用沙箱
- 多线程能力
- 音频能力
- NAPI / Native Engine / Native 模块接入
- 打包、签名、真机调试、性能分析
